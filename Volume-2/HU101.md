## Course Description

HU101 develops the written and spoken communication skills engineers need but rarely receive formal training in: clear documentation, structured technical reports, and confident presentation. It exists because strong technical work with poor documentation is functionally invisible to teammates, employers, and — later in the program — capstone evaluators.

## Why This Course Is Needed
Every lab report, README, and project write-up from this point forward assumes competent technical writing. Without early, dedicated instruction, students develop inconsistent, unclear documentation habits that compound into real friction in team-based courses (SE201, PR301) and the Capstone.

None.

- **PR401/PR402 (Capstone)** — the final report structure assumes fluency in the documentation practices taught here.

## Learning Objectives

1. Write clear, well-structured technical documentation (READMEs, comments, short reports).
2. Deliver a organized, confident technical presentation.
3. Apply basic citation practices and understand academic integrity standards.
4. Adapt communication style to audience (technical peer vs. non-technical stakeholder).

## Learning Outcomes (Assessable)

| # | Outcome | Mapped Objective |
|---|---|---|
| LO1 | Produce a README that clearly explains a program's purpose, setup, and usage | 1 |

## Weekly Topics / Units

| Week | Topic |
|---|---|
| 1 | Why technical communication matters; audience awareness |
| 2 | Writing clear README files: structure, tone, essential sections |
| 3 | Code comments and docstrings: what to document and what not to |
| 4 | Structuring a short technical report: problem, method, result, discussion |
| 5 | Clarity and concision: eliminating jargon and ambiguity |
| 6 | Data and results presentation: tables, simple charts, describing findings in prose |
| 7 | Academic integrity, plagiarism, and citation basics (why it matters in engineering too) |
| 8 | Citation practice: consistent referencing of sources (style TBD by department) |
| 9 | Presentation skills I: structuring a talk, slide design basics |
| 10 | Presentation skills II: delivery, handling Q&A, technical demos |
| 11 | Writing for different audiences: technical peer vs. manager vs. general public |
| 12 | Peer review and editing: giving and receiving constructive written feedback |
| 13 | Revising for clarity: workshop session on student drafts |
| 14 | Final presentation preparation |
| 15 | Final presentations (in-class, graded) |

## Laboratory / Workshop Sessions
Weekly writing workshop pairing with CS101's mini-project — students document that project's README as recurring practice material across the semester.

## Assignments
- 3 short writing assignments (README, short technical report, audience-adaptation rewrite).
- 1 citation exercise.

## Mini Projects
- **Final Presentation (Week 15):** 5-minute individual presentation on the CS101 mini-project (shared artifact across courses), assessed on structure, clarity, and Q&A handling.
