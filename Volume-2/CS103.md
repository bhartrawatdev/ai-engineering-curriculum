## Course Description

CS103 is the applied-practice partner to CS101. Where CS101 delivers concepts through lecture, CS103 exists purely to build muscle memory: converting theory into working code under guided, then independent, conditions. It shares its weekly structure with CS101 by design — there is no separate topic list, because CS103 has no content of its own beyond application.

## Why This Course Is Needed

Programming is a practiced skill, not a memorized one. Separating "lab" from "lecture" as distinct credit-bearing components (rather than folding lab hours into CS101) signals to students that hands-on practice is not optional supplementary work — it is graded, structured, and essential.

## Prerequisite Knowledge Required
None, but must be taken alongside CS101 — the two are functionally inseparable.

## Where It Is Used Later
Directly feeds the CS101 Mini Project (shared deliverable). The debugging and testing habits built here are the assumed baseline for CS105 (Data Structures I) and every subsequent lab course.

---

## Learning Objectives

1. Translate CS101 lecture concepts into working code without guidance.
2. Build an end-to-end command-line program independently.
3. Practice systematic debugging and basic unit testing.

## Learning Outcomes (Assessable)

| # | Outcome | Mapped Objective |
|---|---|---|
| LO1 | Complete weekly coding exercises matching CS101's current unit with no more than one hint request | 1 |
| LO2 | Build a working multi-function program from a written spec, unaided | 2 |
| LO3 | Write at least one `assert`-based test per function in a submitted program | 3 |

## Weekly Structure

CS103 mirrors CS101's 15-week schedule exactly (see `Volume-2/CS101.md` Weekly Topics for full unit list). Each week's lab session is documented in `Volume-3/Labs/CS101-Lab-Manual.md`, which serves as the shared lab manual for both CS101 and CS103 — CS103's grade is calculated from the same lab performance data, weighted differently (see Assessment Scheme below).

## Assessment Scheme

| Component | Weight |
|---|---|
| Weekly Lab Exercises (Weeks 1–13) | 50% |
| Assignments (shared with CS101, see `Volume-4/Assignments/CS101-Assignments.md`) | 20% |
| Mini Project (shared deliverable, lab-execution component) | 30% |

**Passing threshold:** 40% overall.

**Note on shared grading:** CS101 and CS103 share exercises, assignments, and the mini-project artifact, but are graded as separate line-items on the transcript with different weightings — CS101 emphasizes conceptual/exam performance, CS103 emphasizes lab execution and independent practice.
