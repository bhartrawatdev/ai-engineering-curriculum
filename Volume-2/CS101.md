## Course Description

CS101 is the entry point of the entire curriculum. It introduces programming not as syntax memorization but as **computational thinking**: the ability to break a problem into precise, executable steps. Students learn Python as a vehicle for this thinking, building the foundation every later course — data structures, algorithms, machine learning, systems — depends on. By the end of the course, students should be comfortable reading a written problem, planning a solution on paper, and implementing it as a tested, working program.

---

## Why This Course Is Needed

Every course after this one — from CS104 (OOP) to AI201 (Machine Learning) to AI401 (LLMs) — assumes the student can write, read, and debug code fluently. Without this foundation, later abstractions (classes, algorithms, models) become unreachable. This course exists to remove that bottleneck early and permanently.

## Prerequisite Knowledge Required

None. This is an intentional zero-prerequisite entry course.

## Where It Is Used Later

- **CS103** (same semester) — applies these concepts in lab form.
- **CS104/CS105** (Semester 2) — Object-Oriented Programming and Data Structures assume fluent Python.
- **AI201 onward** — every ML/DL course requires this level of programming fluency as a baseline.
- **SE201** (Software Engineering Principles) builds directly on the functions/modularity habits formed here.

---

## Learning Objectives

By the end of this course, students will be able to:

1. Explain and apply the core building blocks of programming: variables, types, control flow, and functions.
2. Decompose a problem statement into a step-by-step algorithm before writing code.
3. Implement, test, and debug Python programs independently.
4. Read and interpret runtime errors and tracebacks to fix bugs efficiently.
5. Apply basic recursion to solve simple problems.

## Weekly Topics / Units — Full Subtopic Breakdown

### Week 1 — What Is Computation?
- What is a program? Instructions, execution, the fetch-interpret cycle (conceptual, no ASM)
- Installing Python, setting up an interpreter/IDE
- The Python REPL: interactive execution vs. running a `.py` file
- `print()` and basic output
- Comments (`#`) and why documentation starts on day one
- Syntax errors vs. logical errors (first exposure)

### Week 2 — Variables & Data Types
- Variable assignment and naming conventions
- Primitive types: `int`, `float`, `str`, `bool`
- Dynamic typing (Python doesn't require declared types) vs. static typing (conceptual contrast)
- Type checking with `type()`
- Type conversion / casting (`int()`, `str()`, `float()`, `bool()`)
- `None` as an explicit "no value" type
- Basic input handling with `input()`

### Week 3 — Operators & Expressions
- Arithmetic operators: `+ - * / // % **`
- Operator precedence and parentheses
- Comparison operators: `== != > < >= <=`
- Logical operators: `and`, `or`, `not`
- Short-circuit evaluation
- Compound assignment operators (`+=`, `-=`, etc.)
- Expressions vs. statements (conceptual distinction)

### Week 4 — Conditionals
- `if` statement structure and indentation rules
- `if / elif / else` chains
- Nested conditionals
- Truthy and falsy values in Python
- Boolean expressions as conditions
- Ternary (conditional) expressions
- Common conditional pitfalls (`=` vs `==`, dangling logic)

### Week 5 — Loops & Iteration
- `while` loops and loop invariants (informal)
- `for` loops and the `range()` function
- Iterating over strings, lists (preview)
- `break` and `continue`
- Infinite loops and how to avoid/debug them
- Nested loops
- Loop counters and accumulator patterns (sum, count, max/min tracking)

### Week 6 — Strings
- String creation, immutability
- Indexing and negative indexing
- Slicing (`[start:stop:step]`)
- Concatenation and repetition (`+`, `*`)
- Common methods: `.lower()`, `.upper()`, `.strip()`, `.split()`, `.join()`, `.replace()`, `.find()`
- String formatting: f-strings, `.format()`, `%` (legacy, brief mention)
- Escape characters and multi-line strings

### Week 7 — Lists & Tuples
- List creation, indexing, slicing (parallels string slicing)
- Mutability (lists) vs. immutability (tuples) — why the distinction matters
- List methods: `.append()`, `.insert()`, `.remove()`, `.pop()`, `.sort()`, `.reverse()`
- List comprehensions (introductory level)
- Nested lists (2D structure, conceptual precursor to matrices in MA104)
- Tuple packing/unpacking
- Iterating over lists with `for`

### Week 8 — Dictionaries & Sets
- Dictionary creation, key-value pairs
- Accessing, adding, updating, deleting entries
- `.keys()`, `.values()`, `.items()`
- Iterating over a dictionary
- Sets: creation, uniqueness property
- Set operations: union, intersection, difference
- When to use dict vs. list vs. set (decision-making, not just syntax)

### Week 9 — Functions
- Why functions: eliminating duplication, abstraction
- Function definition syntax, parameters, return values
- Default parameter values
- Positional vs. keyword arguments
- Variable scope: local vs. global
- `None` as an implicit return
- Docstrings and function documentation conventions
- Pure functions vs. functions with side effects (conceptual introduction)

### Week 10 — Recursion
- What is recursion? Base case and recursive case
- Call stack visualization (how recursive calls are tracked)
- Simple recursive problems: factorial, sum of a list, digit sum
- Recursion vs. iteration — trade-offs (informal, deepened later in CS202)
- Common recursion bugs: missing base case, infinite recursion
- Stack overflow (conceptual, not deep OS-level detail — that's CS204)

### Week 11 — File I/O & Exception Handling
- Opening/closing files: `open()`, `with` statement (context managers, conceptual)
- Reading modes: `.read()`, `.readline()`, `.readlines()`
- Writing to files: `write()`, append vs. overwrite modes
- Introduction to exceptions: `try / except`
- Common exception types: `ValueError`, `TypeError`, `FileNotFoundError`, `ZeroDivisionError`
- Raising exceptions with `raise` (introductory)

### Week 12 — Debugging Methodology
- Reading a Python traceback top-to-bottom
- Isolating the failure point (binary search debugging)
- Print-debugging strategy and its limits
- Introduction to a debugger (breakpoints, step-over, step-into) — tool-agnostic, reinforced practically in CS102
- Rubber-duck debugging as a practice habit
- Common beginner bug categories: off-by-one errors, type mismatches, mutable default arguments

### Week 13 — Code Style, Documentation & Intro to Testing
- PEP 8 basics: naming, whitespace, line length
- Writing meaningful variable/function names
- Docstrings vs. inline comments — when to use which
- Why testing matters (conceptual motivation, not a full testing course — that's SE201)
- `assert` statements as lightweight sanity checks
- Writing a simple manual test case for a function

### Week 14 — Integration Week
- Applying Weeks 1–13 together in the mini-project
- Planning: breaking the mini-project spec into functions before coding
- Instructor/TA-guided project work session
- Peer code review (informal, first exposure — formalized later in SE201)

### Week 15 — Review & Buffer
- Cumulative review of all units
- Common exam-style problem walkthroughs
- Buffer time for makeup labs or extended project work
- Course-level retrospective: connecting back to "Where This Is Used Later"

## Assignments

*(Full detail expands further in Phase 6 — summary below)*

- 5 short problem-set assignments (auto-graded where possible), roughly biweekly.
- Each assignment tests the current week's concepts plus cumulative review.

## Mini Projects

- **Mini Project (Week 14–15):** A command-line program of the student's choosing from an approved list (e.g., a simple text-based game, a unit converter, a basic contact book) that must use: functions, at least one loop, at least one conditional branch, and file I/O.

## Semester Project Contribution

CS101 does not have a standalone semester project (see PR301 in Semester 6), but the mini-project's code and habits (functions, testing, documentation) are the baseline expected in CS103's lab project and all future project courses.
